# Описание датасетов

Документация описывает три файла:

- `incity.csv` — детальные данные по заказам и тендерам.
- `pass_detail.csv` — дневные метрики пассажиров.
- `driver_detail.csv` — дневные метрики водителей.

# `incity.csv` — детальные заказы и тендеры

Файл содержит строки с событиями по городским заказам. Одна строка соответствует одной комбинации заказа и тендера. Если `tender_id` пустой, строка описывает заказ без конкретного тендера.

## Общие поля

- `city_id` — идентификатор города.
- `offset_hours` — смещение локального времени города относительно UTC в часах.
- `order_id` — идентификатор заказа.
- `tender_id` — идентификатор тендера.
- `user_id` — идентификатор пассажира.
- `driver_id` — идентификатор водителя.

## Статусы

- `status_order` — итоговый статус заказа.
- `status_tender` — статус тендера или процесса подбора водителя.

## Временные поля

- `order_timestamp` — время создания заказа.
- `tender_timestamp` — время создания или начала тендера.
- `driveraccept_timestamp` — время принятия заказа водителем.
- `driverarrived_timestamp` — время прибытия водителя к пассажиру.
- `driverstarttheride_timestamp` — время начала поездки.
- `driverdone_timestamp` — время завершения поездки водителем.
- `clientcancel_timestamp` — время отмены заказа пассажиром.
- `drivercancel_timestamp` — время отмены заказа водителем.
- `order_modified_local` — время последнего изменения заказа в локальном времени.
- `cancel_before_accept_local` — время отмены заказа до принятия тендера, если такая отмена была.

## Метрики поездки

- `distance_in_meters` — расчетное расстояние поездки в метрах.
- `duration_in_seconds` — расчетная длительность поездки или заказа в секундах.
- `price_order_local` — итоговая стоимость заказа в локальной валюте.
- `price_tender_local` — стоимость на этапе тендера в локальной валюте.
- `price_start_local` — стартовая стоимость заказа для пассажира в локальной валюте.

## Как интерпретировать данные

- Заказ может иметь несколько строк, если по нему было несколько тендеров.
- `order_id` связывает строки одного заказа.
- `tender_id` описывает конкретное предложение или попытку водителя.
- `user_id` связывает заказ с пассажиром.
- `driver_id` связывает заказ с водителем.
- Временные поля отражают последовательность событий внутри заказа.

# `pass_detail.csv` — дневные метрики пассажиров

Файл содержит агрегированные показатели активности пассажиров по дням. Одна строка соответствует одному пассажиру в одном городе за один день.

## Идентификаторы и даты

- `city_id` — идентификатор города.
- `user_id` — идентификатор пассажира.
- `order_date_part` — локальная дата, за которую посчитаны метрики пассажира.
- `user_reg_date` — дата регистрации пассажира.

## Счетчики заказов

- `orders_count` — количество уникальных заказов пассажира за день.
- `orders_cnt_with_tenders` — количество заказов пассажира, по которым были тендеры.
- `orders_cnt_accepted` — количество заказов пассажира, которые были приняты водителями.
- `rides_count` — количество завершенных поездок пассажира.
- `client_cancel_after_accept` — количество отмен пассажиром после принятия заказа водителем.

## Агрегированные длительности

- `rides_time_sum_seconds` — суммарная длительность поездок пассажира за день в секундах.
- `online_time_sum_seconds` — суммарное время онлайн в секундах.

## Как интерпретировать данные

- `user_id` связывает дневные метрики пассажира с его заказами из `train.csv`.
- `order_date_part` используется для анализа активности пассажиров по дням.
- Метрики подходят для группировок по городу, дате и пассажиру.

# `driver_detail.csv` — дневные метрики водителей

Файл содержит агрегированные показатели активности водителей по дням. Одна строка соответствует одному водителю в одном городе за один день.

## Идентификаторы и даты

- `city_id` — идентификатор города.
- `driver_id` — идентификатор водителя.
- `tender_date_part` — локальная дата, за которую посчитаны метрики водителя.
- `driver_reg_date` — дата регистрации водителя.

## Счетчики заказов и тендеров

- `orders` — количество заказов, связанных с водителем за день.
- `orders_cnt_with_tenders` — количество заказов с тендерами.
- `orders_cnt_accepted` — количество принятых заказов.
- `rides_count` — количество завершенных поездок водителя.
- `client_cancel_after_accept` — количество отмен пассажиром после принятия заказа водителем.

## Агрегированные длительности

- `rides_time_sum_seconds` — суммарная длительность поездок водителя за день в секундах.
- `online_time_sum_seconds` — суммарное время водителя онлайн за день в секундах.

## Как интерпретировать данные

- `driver_id` связывает дневные метрики водителя с его заказами из `train.csv`.
- `tender_date_part` используется для анализа активности водителей по дням.
- Метрики подходят для группировок по городу, дате и водителю.

# Связи между файлами

- `incity.csv.user_id` = `pass_detail.csv.user_id` — связь заказов с дневными метриками пассажира.
- `incity.csv.driver_id` = `driver_detail.csv.driver_id` — связь заказов с дневными метриками водителя.
- `incity.csv.order_timestamp` можно сопоставлять с `pass_detail.csv.order_date_part` для анализа заказов пассажиров по дням.
- `incity.csv.tender_timestamp` можно сопоставлять с `driver_detail.csv.tender_date_part` для анализа активности водителей по дням.
