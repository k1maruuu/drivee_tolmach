# 🚀 Drivee NL2SQL Platform — Team Unicorn

## 📌 Описание проекта

Мы разработали прототип интеллектуальной платформы, которая позволяет бизнес-пользователям получать данные из базы данных без знания SQL, используя естественный язык.

💡 Идея проста:

```text
Спросил → Получил данные → Принял решение
```

Пользователь вводит вопрос на обычном языке, система автоматически:

* понимает смысл запроса
* генерирует SQL
* безопасно выполняет его
* возвращает результат в виде таблицы и графика

Это устраняет разрыв между бизнесом и данными и снижает зависимость от технических специалистов.

---

## ⚙️ Как это работает

### 🔄 Основной pipeline

```text
1. Пользователь вводит вопрос
2. Система проверяет шаблоны (быстрый ответ без ИИ)
3. Если нет шаблона → используется LLM (Ollama qwen3:4b)
4. Генерируется SQL
5. SQL проходит guardrails (безопасность + проверка)
6. Выполняется в PostgreSQL
7. Возвращается результат + объяснение + визуализация
```

---

### 🧠 Важные компоненты

#### 1. Semantic Layer

* переводит бизнес-язык в SQL
* понимает:

  * “отмены” → `status_order = 'cancel'`
  * “выручка” → `SUM(price_order_local)`
* учитывает NULL-значения и бизнес-логику

#### 2. Hybrid NL → SQL

* сначала шаблоны (`goodprompts.txt`)
* потом LLM (если нет совпадения)

#### 3. Guardrails

* блокирует опасные SQL:

  * DROP / DELETE / UPDATE
* проверяет через `EXPLAIN`
* ограничивает тяжёлые запросы
* автоматически добавляет LIMIT

#### 4. Explainability

Система объясняет:

* как поняла вопрос
* какие метрики использовала
* какой SQL сгенерировала

#### 5. Visualization

Backend сразу отдаёт:

* тип графика
* конфиг для фронта

#### 6. Отчёты

* сохранение
* повторный запуск
* расписание

## 🛡️ Безопасность

Система защищена от:

```text

утечки данных
тяжёлых запросов

```

